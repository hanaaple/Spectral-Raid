Spectral Raid

제한된 시야와 소리 기반 정보로 위험을 판단해 탈출을 결정하는
5~8분 세션 기반 탑다운 전술 액션 게임.

Extraction-lite Roguelite 싱글플레이 (멀티 확장 고려) Unity URP 포트폴리오
01

게임 개요

핵심 차별점
강하게 제한된 시야 + 사운드 중심 정보 인지
세션 길이
5 ~ 8분 / 세션
세션 루프
탐색 → 전투/레벨업 → 루팅 → 탈출 판단
세션 간 성장
Soul Shard로 영구 업그레이드·스킬 해금
💡 포트폴리오 방향 원칙 — 설계 결정에는 기술적 근거와 이유가 명확히 드러나야 한다. 단, 과설계로 실제 게임 완성을 저해하는 것은 지양한다.

컨트롤러·캐릭터 분리 (Possession 패턴)

입력을 소유하는 Controller와 실제 이동·전투 로직을 갖는 Character를 분리해 멀티플레이 확장과 AI 전환을 대비한다.

ControllerBase (abstract)       CharacterBase (abstract)
└── PlayerController                    ├── PlayerCharacter
    - _possessedCharacter               └── MonsterCharacter
    - Input 구독 / 해제

ControllerBase

Possess(CharacterBase), HasAuthority() — 현재 항상 true, 멀티플레이 대비 TODO

PlayerController

OnEnable / OnDisable에서 PlayerInputActions 구독·해제. 입력을 Character 메서드 호출로 전달.

📌 이동 로직은 Controller가 아닌 Character가 소유한다. (PlayerCharacter.MoveTo(Vector3))

AbilitySystem — GAS-like

UE5 Gameplay Ability System(GAS) 개념을 Unity에 맞게 경량화한 수치·이펙트 관리 시스템. Assets/Scripts/Core/AbilitySystem/

🎯 설계 의도 — 장비·스킬·디버프 등 수십 개의 출처가 수치를 동시에 변경할 때, 각 시스템이 필드를 직접 조작하면 적용 순서 의존성과 잔류 값 문제가 발생한다. GAS-like는 모든 수치 변경을 GameplayEffect 단일 채널로 추상화해 적용·해제가 서로 독립적으로 동작하도록 보장한다.

BaseValue / CurrentValue 이중 구조

BaseValue는 영구 원본값, CurrentValue는 활성 GE 모디파이어를 누산한 결과값이다. UI·전투 판정은 CurrentValue를 읽고, Effect 해제 후에는 BaseValue만 남는다.

📌 dirty-on-write 전략 — CurrentValue는 Effect 적용·해제 시점에만 재계산한다. 읽기 호출 빈도와 무관하게 비용이 없으며, 매 읽기마다 활성 Effect를 전부 순회하는 recalculate-on-read와의 핵심 차이다.
AttributeHandle — Reflection 비용 격리

GE Modifier가 대상 어트리뷰트를 지정하는 경량 식별자. static readonly로 선언해 앱 수명 동안 Reflection(FieldInfo)을 단 1회만 수행하고, 이후 런타임 읽기·쓰기는 캐싱된 참조만 사용한다. 런타임 string 탐색 비용 없음.

SO 초기화 흐름 — 코드 없이 초기 스탯 설정

캐릭터마다 다른 초기 스탯을 코드 변경 없이 Inspector에서 관리한다. SO에 정의된 fieldName·BaseValue 목록을 Awake()에서 읽어 AttributeSet 인스턴스를 생성한다.

GameplayEffect 시스템
에셋 / 런타임 레이어 분리

GE는 불변 정의(ScriptableObject 에셋)와 적용 인스턴스(런타임 Spec)로 철저히 분리된다. SO는 여러 캐릭터가 공유해도 상태가 섞이지 않고, Spec은 적용 시점의 Level·Context를 스냅샷해 독립적으로 관리된다.

📌 Spec 생성 시 Modifier를 미리 resolve하는 이유 — GameplayModifier는 대상 어트리뷰트를 문자열로 직렬화한다. 이를 AttributeHandle(FieldInfo 캐싱)로 변환하는 비용이 있으므로 Spec 생성 시점에 단 1회만 수행한다. GE SO 하나가 수십 개의 Active Effect로 동시에 적용되더라도 resolve 비용은 각 Spec마다 1회뿐이다.
GameplayEffectType — Instant / Infinite / Duration
Instant
BaseValue를 즉시 영구 변경 후 종료. Active Effect로 등록되지 않으므로 Handle 없음.
예: 체력 회복, 데미지
Infinite
명시적 Remove 전까지 CurrentValue에 지속 반영.
예: 장비 스탯 버프, 패시브
Duration
지정 시간 후 자동 해제. Period 설정 시 주기마다 추가 실행.
예: 버프·디버프, DoT·HoT
Apply 흐름

ApplyGameplayEffectToSelf는 GE SO로부터 Spec(런타임 인스턴스)을 생성하고 타입에 따라 분기한다. Instant는 즉시 BaseValue를 변경 후 Handle 없이 종료. Duration / Infinite는 _activeEffects에 등록하고 Handle을 반환 — 핸들로 명시적 해제 가능.

⏱ Period > 0이면 매 프레임 경과 시간을 누적하고 주기 도달 시 BaseValue를 직접 변경한다. CurrentValue를 수정하는 persistent modifier와 달리 원본 수치를 반복 수정하므로 DoT·HoT에 적합하다.
ModifierOperation — CurrentValue 계산

6가지 연산(AddBase, MultiplyAdditive, DivideAdditive, MultiplyCompound, AddFinal, Override)을 조합해 활성 GE 전체를 한 번의 aggregation으로 처리한다.

CurrentValue = ((Base + ΣAddBase) × MultiplyAdditive / DivideAdditive × ΠMultiplyCompound) + ΣAddFinal

Override가 하나라도 있으면 위 계산 전체를 무시하고 해당 값으로 고정. — 예: 스턴 시 Speed = 0

GameplayEffectContext — 발동 맥락

GE 적용 시 발동 주체(Instigator ASC)와 실제 원인 오브젝트(EffectCauser)를 함께 전달한다. GameplayEffectContextHandle은 null-safe struct 래퍼로, 맥락이 불필요하면 default로 생략 가능하다. Execution 내부에서 Source·Target의 Context를 읽어 크리티컬·관통 등 조건부 계산에 활용할 수 있다.

GameplayEffectExecution — 복합 계산 서브클래스 없음
🔧 단순 Modifier로 표현할 수 없는 복합 계산 (예: Source.Damage − Target.Armor × 관통률)을 GameplayEffectExecution ScriptableObject 서브클래스로 캡슐화한다. Execute(parameters) 안에서 Source·Target ASC를 읽고 parameters.AddOutput(handle, magnitude)으로 결과를 출력하면 ASC가 AddBase / Override 연산으로 BaseValue에 즉시 반영한다.
